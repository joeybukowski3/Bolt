📄 Methodology

How we determine manufacture dates from serial numbers.

Overview

Serial Number Decoder uses a curated database of manufacturer serial format rules compiled from publicly available technical documentation, service manuals, and trusted industry references. Each supported brand has a defined decoding method that extracts date information embedded within the serial number by the manufacturer.

This is a rules-based system. When a serial number is entered, we apply the brand-specific format rules and return the encoded date without any estimation or inference. The output reflects what the manufacturer encoded in the serial number itself.

How It Works

  1. 1 The user selects the brand and enters the full serial number as shown on the product label.
  2. 2 The decoder applies the brand's documented format rules to extract the manufacture date.
  3. 3 The result is returned immediately — no external API calls are made for the serial decode path.
  4. 4 A confidence level is assigned based on whether the result is a single unambiguous year or a dual-decade result (see below).

Confidence Levels

High — The serial number returned a single, unambiguous manufacture year with no decade ambiguity.

Medium — The brand's serial format reuses year codes on a repeating cycle (commonly 20–30 years), resulting in two possible valid years. Physical inspection or supporting documentation is needed to confirm the correct decade.

Low — The serial number could not be decoded, or the brand (e.g., Kenmore) sources from multiple OEM manufacturers with different format rules. Additional information is required.

Alternative Lookup (AI-Assisted)

When no serial number is available, our Alternative Lookup section accepts a model number or free-text description. This path uses an AI language model to estimate the product's manufacture era based on publicly known product history. Results from this path carry lower certainty than direct serial decoding and should be treated as estimates only.
